OC 入门

OC 是 C 语言的超集完全兼容 C 语言的所有语法, 本文主要关注 OC 中新添加的一些属性

Hello World

#import

import 是 OC 中引入的新预编译命令,用来代替 C 语言中 include ,其优点是无论该头文件被引用多少次, 编译的时候只包含一次. (include只是简单复制不做这个检查. )

在 C 语言中，程序员通常使用基于 #ifdef 命令的方案来避免一个文件包含另一个文件，而后者又包含第一个文件的情况。

而在 Objective-C 中，程序员使用 #import 来实现这个功能。

Fundation 框架

Foundation 框架处理的是用户界面之下的层（layer）中的特性，例如数据结构和通信机制。IOS开发中几乎所有程序都是以 Foundation 框架为基础。如果你想知道 Foundation 框架包含哪些头文件，可以查看其 Header 目录（/System/Library/Frameworks/Foundation.framework/Headers/）。

NSLog() 和 @“字符串”

NSLog 作用类似于 C 语言的printf() ，调用方法也很类似。但我们建议用 NSLog() ，因为它添加了特性，例如时间戳、日期戳和自动附加换行符（\n）等。

@ 符号是 Objective-C 在标准 C 语言基础上添加的特性之一。双引号中的字符串前有一个 @ 符号，这表示引用的字符串应该作为 Cocoa 的 NSString 元素来处理。NSString 元素封装了许多特性，Cocoa 在需要字符串时可以随时使用它们。下面是一些 NSString 的功能。

  • 告知其长度；
  • 将自身与其他字符串比较；
  • 将自身转换为整型值或浮点值。

BOOL 与 boolean

在 C99 发布前, C 语言中没有严格的 Boolean类型, 程序员通常 用 零和 非零 表示flase和true. 这样虽然增强了 程序实现的灵活性 , 但同时降低了可读性 和 可维护性 . 造成一些难以理解的 Bug . 为了 改善这个问题 OC 引入的新的书记类型 BOOL 来表示开关量. 通过显示赋值来降低Magic Bug的发生.
